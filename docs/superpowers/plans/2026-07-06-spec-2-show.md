# Spec 2 — Show (`GET /posts/{id}`) Implementation Plan

> **For agentic workers:** implement this plan task-by-task with TDD. Use the existing
> FastAPI/SQLAlchemy/Pytest stack and do not modify frozen Spec 0 files.

**Goal:** Implementar `GET /posts/{id}` para devolver un post con el contrato canónico
de Spec 0, respondiendo `404 POST_NOT_FOUND` cuando el id no existe o cuando el post
está en `trash`.

**Architecture:** El endpoint en `app/routers/posts.py` consulta el post por id usando
`get_db()`. Si no existe o si su `status` es `trash`, lanza `PostNotFound` para que el
handler global de Spec 0 serialice el error estándar. Si existe y no está en `trash`,
devuelve el modelo usando `PostRead` sin wrappers extra.

**Tech Stack:** FastAPI, SQLAlchemy 2.0, Pydantic v2, pytest.

## Global Constraints

- Contrato canónico de Post: 11 campos exactos vía `PostRead`.
- `GET /posts/{id}` responde `404 POST_NOT_FOUND` tanto para id inexistente como para
  posts en `trash`.
- No implementar control de acceso por `private`.
- No tocar archivos congelados de Spec 0.
- No introducir librerías nuevas.

## File Structure

- `app/routers/posts.py` — modificar: agregar `GET /posts/{id}`.
- `tests/test_posts_show.py` — crear: tests de integración del endpoint.

### Task 2: Implementación del endpoint show

**Files:**
- Modify: `app/routers/posts.py`

**Interfaces:**
- Consumes: `get_db`, `Post`, `PostRead`, `PostNotFound`.
- Produces: `GET /posts/{id}` con `200` o `404 POST_NOT_FOUND`.

- [ ] **Step 1: Implementar el handler mínimo**

Agregar en `app/routers/posts.py` un handler similar a este flujo:

```python
@router.get("/{id}", response_model=PostRead)
def show_post(id: int, db: Session = Depends(get_db)):
    post = db.get(Post, id)
    if post is None or post.status == "trash":
        raise PostNotFound()
    return post
```

La implementación final puede usar `query(...).filter(...)` o `db.get(...)`, pero debe
mantener la misma regla de negocio: `trash` responde igual que inexistente.
